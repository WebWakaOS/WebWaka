## Purpose

Geography is both a structural model and an aggregation engine. It powers location identity, discovery, filtering, reporting, and automatic marketplace or directory generation.

## Typed Geography Hierarchy

| Level | Type |
|---|---|
| 1 | Country |
| 2 | Geopolitical Zone |
| 3 | State |
| 4 | Local Government Area |
| 5 | Ward |
| 6 | Community |
| 7 | Household / Family |
| 8 | Facility Place |

## Facility Place Examples

- Market
- Motor park
- Warehouse
- Office
- Branch
- Hub
- Constituency office
- Campus
- Clinic
- School

## Key Rules

1. All geography-bearing units are modeled as Places.
2. Every Place may have a parent Place.
3. Ancestry must be queryable.
4. Profiles and Offerings must be indexable by geography.
5. Discovery collections may be generated automatically at any geography level.

## Required Query Support

- All markets in an LGA
- All office holders in a state
- All vendors across the country
- All motor parks in a senatorial zone
