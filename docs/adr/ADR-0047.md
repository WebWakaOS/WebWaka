# ADR-0047: Chaos Engineering Baseline — Wave 3 D5

**Status:** Accepted  
**Date:** 2026-05-02  
**Deciders:** Engineering Team  
**Context:** Wave 3 production readiness gate requires documented chaos scenarios before first production load.

---

## Context

WebWaka relies on three external dependencies whose failure must produce graceful degradation rather than hard failures:

1. **D1 (Cloudflare D1)** — primary datastore for all transactional data
2. **AI Provider (OpenRouter/OpenAI/Anthropic)** — SuperAgent chat, vertical AI advisor
3. **KV (Cloudflare Workers KV)** — rate limits, feature flags, session cache

Each must fail safely: the platform should continue serving degraded-but-functional responses rather than returning 500s or exposing errors to end users.

---

## Decision

Implement chaos scenarios as automated tests that inject failures at the adapter layer and assert degraded-but-functional behaviour. No production traffic is involved. Scenarios run in CI against in-memory stubs.

### Scenario 1: D1 Timeout / Unavailable

**Inject:** D1 `prepare().run()` throws `Error('D1_UNAVAILABLE')`.

**Expected degradation:**
- `/health/deep` returns `200 { status: "down", checks: { d1: { ok: false } } }`
- All entity reads return `503 { error: "database_unavailable", retry_after: 5 }`
- No 500 Internal Server Error leaks raw DB error messages
- Write operations queue to a dead-letter pattern (log + return 503) rather than throwing

**NOT acceptable:**
- Unhandled promise rejection in Worker
- Raw D1 error message surfaced in API response
- Silent data loss

### Scenario 2: AI Provider Down

**Inject:** `fetch()` to AI provider URL throws `TypeError('Failed to fetch')` or returns `503`.

**Expected degradation:**
- SuperAgent chat endpoint returns `200 { message: "AI service temporarily unavailable. Try again shortly.", degraded: true }` (not a 500)
- Vertical AI advisor page shows "AI temporarily unavailable" banner (frontend)
- `/health/deep` returns `200 { status: "degraded", checks: { ai_provider: { ok: false } } }`
- WakaCU is NOT charged for failed AI completions (P9 monetary integrity)
- Fallback model (if configured) is tried before returning degraded response

**NOT acceptable:**
- AI API key leaked in error response
- WakaCU charged for zero-token response
- 500 returned to workspace user

### Scenario 3: KV Unavailable

**Inject:** KV `.get()` / `.put()` throws `Error('KV_UNAVAILABLE')`.

**Expected degradation (ARC-17: fail-open on KV):**
- Rate limiting **fails open** — request is allowed through (conservative default)
- Feature flags fall back to compiled-in defaults
- Session cache miss — re-derives session from JWT (slower but functional)
- `/health/deep` returns `200 { status: "degraded", checks: { kv: { ok: false } } }`

**NOT acceptable:**
- Rate limiter blocks all requests when KV is down (cascading failure)
- Feature flag failure causes complete feature blackout
- 500 returned because KV threw

---

## Consequences

- All three scenarios are tested in `tests/chaos/` (see below)
- New handlers MUST add a chaos test before merging if they use D1, AI, or KV
- `PRODUCTION_READINESS_BACKLOG.md` D5 is resolved by this ADR + tests
- Future chaos scenarios (network latency, Paystack outage, Prembly outage) added incrementally
