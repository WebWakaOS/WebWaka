==================================================
NON-NEGOTIABLE RULES
==================================================

1. BUILD FROM VERIFIED FINDINGS
Use only verified findings from the completed audit/QA cycle.
If a point was marked uncertain or deferred, keep that uncertainty explicit.

2. CURRENT REALITY FIRST
The PRD must begin from the present codebase and current architecture, not from a fantasy greenfield system.

3. PRE-LAUNCH REFACTOR WINDOW
Assume we have not launched yet.
This means:
- deeper refactors are acceptable now,
- deprecations should be done now where beneficial,
- naming/model resets should happen now if necessary.

4. BUILD ONCE, USE INFINITELY
This is a core design doctrine, not a slogan.
The PRD must operationalize it with concrete rules, scope boundaries, and engineering constraints.

5. OFFLINE-FIRST / MOBILE-FIRST / PWA-FIRST
These must be first-class PRD concerns, not side notes.

6. NO VAGUE PHASES
Each phase must have:
- objective,
- scope,
- in-scope modules,
- out-of-scope items,
- dependencies,
- risks,
- acceptance criteria,
- release gate.
